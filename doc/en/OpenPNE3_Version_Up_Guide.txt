How to Update OpenPNE 3.4.x
===========================

Getting Started
---------------

This document is for updating OpenPNE 3.2/OpenPNE 3.4 to OpenPNE 3.4.x.

Updating Steps
--------------

The updating steps are the followings:

(1) Upload source files
(2) Create the configuration files
(3) Back up the database
(4) Execute the updating command

The Way to Work
---------------

Upload source files
~~~~~~~~~~~~~~~~~~~

Upload the OpenPNE 3.4.x files to your web server.

You must upload the source files to a new directory to avoid the troubles by mixing the new files with the old.

Create the configuration files
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Copy "config/ProjectConfiguration.class.php.sample" to "config/ProjectConfiguration.class.php".

Copy "config/OpenPNE.yml.sample" to "config/OpenPNE.yml".

(If you want to add your own configurations to the new files, you should read your old configuration files for reference.)

Copy the old "config/database.yml" to "config/database.yml".

Back up the database
~~~~~~~~~~~~~~~~~~~~

::

  $ mysqldump -u $USERNAME -p --default-character-set=utf8 $DBNAME > /tmp/backup.dump

Execute the Updating Command
~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Execute the following command in the project root directory.

::

  $ ./symfony openpne:migrate

This command will update OpenPNE and plugins to newer version.

Now, updating has been done.
